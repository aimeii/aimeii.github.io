Summary

My team and I tackled improving the user experience for Venmo’s iOS app. We based our redesign choices from the common mistakes that Venmo users made. I iterated on paper prototypes before creating high fidelity mockups.

Skills: User Research, Paper Prototyping, Iterating from Feedback, High Fidelity Prototyping

User Research

We brought in 18 subjects ranging in familiarity with Venmo for user research. In addition to answering interview questions, we had them complete a demo which provided us with crucial data on their habits and behavior. We screen recorded these demos in order to identify the user’s struggles and errors.


Existing Problems

1. Lack of familiarity or interest for non personal Venmo transactions (50% of test users)

2. Paying or requesting the wrong person (25% of test users)

3. Unitentionally paying instead of requesting (25% of test users)

4. Not entering a mandatory caption (31% of test users)

5. Confusion between the complete and incomplete notificaiton tabs (87.5% of test users)


Redesigns and New User Flow

1. Home Page

Problem: Lack of interest and knowledge for non personal transactions

Solution: A new home page focused on the user's personal transaction history

2. Notifications

Problem: Confusion between "Notifications" and "Incomplete" tabs

Solution: A single "Notifications" page with clear categories to differenciate sent and recieved notifications

3. Confirmation

Problem: Users pay or request the wrong person due to lack of identifiable information

Solution: Add a confirmation page to ensure intended target is correct. Enlarge profile picture and include additional identifiable details

4. Select a Recipient

Problem: Users do not use the friends feature since it is hard to locate and unorganized

Solution: Increase size of "Top People" section while decreasing amount of top people and alphabatize the friends list

5. Pay versus Request

Problem: Users will accidentally pay others when they mean to request them

Solution: Start a transaction by declaring to pay or request so accidents are resolved early on

6. Mandatory Caption

Problem: Users forget to put a mandatory caption since there are no signifiers

Solution: Disable action until user provides a caption, they can easily do so by selecting an emoji which represents a category of transaction


Project Conclusion & Takeaways

I learned a lot from this project about human centered design. As opposed to other projects, this project was very data driven. Every redesign decision made had to have statistics to back it up. In order to get this data, we had to conduct our interviews and demos very objectively. Many of our original predictions about common errors users made or features users liked, were proven wrong through research.

After going through this process, I have gained a deeper understanding for one of my favorite quotes from class:

"Art is when we make something for ourself, design is when we make something for others."

This project started in that design class. Thank you to my class project partners Meredith Fan, Isabella Quezada, Angela Quiambao, and Pauline Thach.
